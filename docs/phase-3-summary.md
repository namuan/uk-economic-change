## Objective

Parse the 4 raw ONS source files, extract 2007 baseline and latest available values, and normalise into the canonical skeleton format used by `build_outputs.py`.

---

## What was built

### `src/process_indicators.py` — new module

| Function | Input | Output |
|----------|-------|--------|
| `process_gdp_per_head()` | `data/raw/ihxw.csv` | GDP per head: 2007 & 2025 annual |
| `process_ndp_per_head()` | `data/raw/mwb6.csv` | NDP per head: 2007 & 2025 annual |
| `process_labour_productivity()` | `data/raw/prdy.csv` | Output per hour (LZVB): 2007 & 2025 annual |
| `process_regional_productivity()` | `data/raw/prodbyreg.xlsx` | 12 regions: 2007 & 2023 Output per hour |

Two builder functions assemble the rows into DataFrames matching the skeleton schema, and `write_processed()` writes them to `data/processed/`.

### Data handling details

| Challenge | Solution |
|-----------|----------|
| **Quarterly vs annual** | `_latest_annual()` filters out rows containing "Q" in the period label; selects last annual-only row |
| **Metadata headers** | `skiprows=7` for generator CSVs; `skiprows` callable for PRDY's 6-line header |
| **Multi-sheet XLSX** | Hardcoded `Table_2` for Output per hour; rows 17 (2007) and 33 (2023) |
| **England exclusion** | Filtered out from regional table (it's the UK=100 baseline, not a region in our framework) |
| **Column mapping** | `REGION_MAP` dict translates ONS labels to project geography names |

## Full pipeline (single command chain)

```bash
uv sync                                           # install dependencies
uv run python src/fetch_ons.py                    # download raw ONS data (cached)
uv run python src/process_indicators.py            # normalise into processed tables
uv run python src/build_outputs.py                # compute changes, generate outputs & charts
```

All steps are idempotent and reproducible.
